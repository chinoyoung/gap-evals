A. Grouped Accordion

Add categories to questions. Each category is a collapsible section showing a summary count. Only one group expanded at a time. Includes search bar at top.

Pros

  • Hides complexity — only see what you need
  • Clear mental model of question organization
  • Scales well as library grows

Cons

  • Requires adding a "category" field to each question
  • Extra step when creating questions

B. Compact Card Grid

Replace the flat list with a 2-3 column card grid. Each question is a small card showing type badge and text. Add a search bar. No categories needed — just visual breathing room.

Pros

  • No schema changes — works with existing data
  • More scannable than a vertical list
  • Search bar helps find specific questions

Cons

  • Less structured than categories
  • Cards may feel repetitive at 50+ questions

C. Streamlined List + Search

Keep the list layout but declutter it: remove drag handles, add a search bar, add summary stats at top, tighten row spacing, and show actions only on hover. Minimal changes, maximum impact.

Pros

  • Least effort — refines what exists
  • No schema changes needed
  • Familiar list pattern

Cons

  • Still a flat list — no grouping
  • May not feel like enough of a change
